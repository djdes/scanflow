# Пошаговая инструкция: сборка и использование обработки для 1С:УНФ 1.6

**Что получится в итоге:** внешняя обработка `.epf`, которую открываешь в 1С, нажимаешь "Загрузить" — и все распознанные накладные из `scan.magday.ru` автоматически превращаются в документы "Приходная накладная" с товарами, контрагентами и суммами.

---

## Что уже готово (на стороне сервера)

- **Адрес сервиса:** `http://scan.magday.ru`
- **API-ключ:** `your-secret-api-key` (тот же что в дашборде)
- **Эндпоинты** полностью реализованы и протестированы (32 теста прошли на проде)
- **Исходники 1С** лежат в папке [`1c/`](.) — осталось только собрать в `.epf`

---

## Часть 1. Сборка внешней обработки (один раз)

### Что понадобится
- Установленная 1С:Предприятие 8.3 (клиент-серверная или файловая база УНФ 1.6)
- Доступ к **Конфигуратору** твоей базы УНФ 1.6
- Файлы из папки `1c/`:
  - [`МодульОбъекта.bsl`](МодульОбъекта.bsl) (34 KB)
  - [`МодульФормы.bsl`](МодульФормы.bsl) (12 KB)
  - [`СтруктураФормы.txt`](СтруктураФормы.txt) — справочный, описывает элементы формы

### Шаг 1. Открыть Конфигуратор

1. Запусти 1С:Предприятие (ярлык "1С Предприятие" или "Запуск 1С:Предприятия")
2. В окне выбора информационных баз — **выбери свою базу УНФ 1.6**
3. Внизу окна нажми **Конфигуратор** → **Запуск**
4. Введи логин/пароль администратора

### Шаг 2. Создать новую внешнюю обработку

1. Меню **Файл** → **Новый...** → в диалоге выбери **Внешняя обработка** → **ОК**
2. Откроется дерево новой обработки с пустыми ветками
3. В левой панели кликни **правой кнопкой** на корне (там где "ВнешняяОбработка1") → **Свойства**
4. В палитре свойств справа задай:
   - **Имя:** `ЗагрузкаНакладныхJPGExchange` (без пробелов)
   - **Синоним:** `Загрузка накладных из JPGExchange`
   - **Комментарий:** `Загружает распознанные накладные со scan.magday.ru`

### Шаг 3. Добавить реквизиты обработки

Реквизиты обработки — это её "поля", которые будут отображаться на форме.

В дереве обработки найди ветку **Реквизиты** → правой кнопкой → **Добавить**.

Добавить **6 реквизитов** (для каждого повторить: правой кнопкой → Добавить → задать свойства):

| № | Имя | Тип | Длина |
|---|-----|-----|-------|
| 1 | `АдресСервиса` | Строка | 200 |
| 2 | `КлючAPI` | Строка | 100 |
| 3 | `Организация` | СправочникСсылка.Организации | — |
| 4 | `Склад` | СправочникСсылка.Склады (или СправочникСсылка.СтруктурныеЕдиницы) | — |
| 5 | `СоздаватьНоменклатуру` | Булево | — |
| 6 | `ПроводитьДокументы` | Булево | — |

> **Важно для реквизита 4 "Склад":** в разных версиях УНФ 1.6 склад называется по-разному. Попробуй сначала `СправочникСсылка.Склады`. Если такого справочника нет — используй `СправочникСсылка.СтруктурныеЕдиницы`. Если и такого нет — оставь тип `Строка(100)` и вручную будешь вводить название.

### Шаг 4. Вставить код модуля объекта

Это основной код, который делает HTTP-запросы и создаёт документы.

1. В дереве обработки кликни **дважды на корне** (`ЗагрузкаНакладныхJPGExchange`) — откроется модуль объекта (пустое окно кода)
   - Или нажми `Ctrl + Alt + M`
2. Открой файл [`МодульОбъекта.bsl`](МодульОбъекта.bsl) в блокноте
3. **Выдели весь текст** (`Ctrl + A`), **скопируй** (`Ctrl + C`)
4. В Конфигураторе, в окне модуля объекта → **вставь** (`Ctrl + V`)
5. **Сохрани** (`Ctrl + S`)

### Шаг 5. Создать форму обработки

1. В дереве обработки правой кнопкой на ветке **Формы** → **Добавить**
2. Откроется конструктор форм:
   - **Тип формы:** `Форма обработки`
   - **Имя формы:** `Форма` (или `ФормаОбработки`)
   - **Синоним:** `Загрузка накладных`
   - Галочка **Назначить основной** → ✅
3. Нажми **Готово**

Откроется редактор формы с пустой формой.

### Шаг 6. Разместить элементы на форме

В редакторе формы слева — дерево элементов, справа — область разметки. Будем перетаскивать реквизиты из панели "Реквизиты" в "Элементы".

**6.1. Группа "Настройки подключения"**

1. На вкладке **Элементы** правой кнопкой → **Добавить** → **Группа — Обычная группа**
2. В свойствах группы:
   - Имя: `ГруппаНастройки`
   - Заголовок: `Настройки подключения`
   - Отображение: `Обычное выделение`
3. Перетащи в эту группу из вкладки **Реквизиты** → `Объект.АдресСервиса`
4. Перетащи в эту же группу `Объект.КлючAPI`. В свойствах созданного элемента задай **Режим пароля = Да** (скроет символы звёздочками)

**6.2. Группа "Параметры загрузки"**

1. Добавь ещё одну обычную группу: `ГруппаПараметры` с заголовком `Параметры загрузки`
2. Перетащи туда:
   - `Объект.Организация`
   - `Объект.Склад`
   - `Объект.СоздаватьНоменклатуру`
   - `Объект.ПроводитьДокументы`

**6.3. Группа кнопок**

1. Добавь обычную группу `ГруппаКоманды`. В свойствах:
   - Заголовок: оставь пустым
   - **Группировка:** `Горизонтальная`

Кнопки создаются из **Команд**, а команды мы добавим на следующем шаге.

**6.4. Поле лога**

1. Вкладка **Реквизиты** (реквизиты **формы**, не обработки!) → правой кнопкой → **Добавить**
2. Свойства:
   - Имя: `ПолеЛога`
   - Тип: **Строка**
   - Длина: **0** (неограниченная)
3. Перетащи `ПолеЛога` из реквизитов на форму под группу кнопок
4. В свойствах появившегося элемента:
   - **Многострочный режим:** Да
   - **Только просмотр:** Да
   - **Высота:** 15 (примерно)
   - **Заголовок:** `Журнал операций`
   - **РастягиватьПоВертикали:** Да

### Шаг 7. Добавить команды формы

Команды — это действия кнопок.

1. В редакторе формы перейди на вкладку **Команды**
2. Правой кнопкой → **Добавить** → создай **4 команды**:

| Имя команды | Заголовок кнопки |
|-------------|------------------|
| `КомандаЗагрузить` | `Загрузить накладные` |
| `КомандаПроверитьСоединение` | `Проверить соединение` |
| `КомандаСохранитьНастройки` | `Сохранить настройки` |
| `КомандаОчиститьЛог` | `Очистить лог` |

3. Для каждой команды в свойствах — **Действие** (иконка лупы) → появится заготовка процедуры в модуле формы → **Отмена** (не создавай пустую, мы вставим свой код)

### Шаг 8. Разместить кнопки в группе команд

1. Перетащи каждую команду из вкладки **Команды** в элемент формы `ГруппаКоманды` — получится кнопка
2. В результате в группе должно быть 4 кнопки в ряд

### Шаг 9. Вставить код модуля формы

1. Внизу редактора формы — вкладка **Модуль** (или кнопка `Модуль формы`)
2. В этом модуле будет заготовка, возможно с пустыми процедурами команд
3. **Удали всё** что там есть (`Ctrl + A`, `Delete`)
4. Открой [`МодульФормы.bsl`](МодульФормы.bsl), скопируй всё содержимое и вставь в модуль формы
5. **Сохрани** (`Ctrl + S`)

### Шаг 10. Сохранить как .epf

1. Меню **Файл** → **Сохранить как...**
2. В диалоге сохранения:
   - **Тип файла:** `Внешние обработки (*.epf)`
   - **Имя файла:** `ЗагрузкаНакладныхJPGExchange.epf`
   - Выбери папку куда сохранить (например, `C:\1C\Обработки\`)
3. **Сохранить**

Готово — у тебя есть `.epf` файл.

### Шаг 11. Проверка в режиме 1С:Предприятие

1. Закрой Конфигуратор (сохрани если спросит)
2. Запусти 1С:Предприятие — зайди в свою базу УНФ **в обычном режиме** (не Конфигуратор)
3. Меню **Файл** → **Открыть...** → укажи путь к `ЗагрузкаНакладныхJPGExchange.epf`
4. Должна открыться форма с полями "Адрес сервиса", "API-ключ" и кнопками

> Если 1С ругается на безопасность внешних обработок — в Администрировании → Общие настройки → разрешить открытие внешних обработок.

## Часть 2. Использование (каждый раз когда нужно загрузить накладные)

### Шаг 1. Настройка (первый раз)

1. Открой обработку: **Файл** → **Открыть** → `ЗагрузкаНакладныхJPGExchange.epf`
2. Заполни поля:
   - **Адрес сервиса:** `http://scan.magday.ru`
   - **API-ключ:** `your-secret-api-key` (можно посмотреть/поменять в [настройках дашборда](http://scan.magday.ru/#/settings))
   - **Организация:** выбери из справочника свою организацию
   - **Склад:** выбери склад куда приходят товары
   - ✅ **Создавать номенклатуру автоматически** (иначе товары без соответствия в 1С будут пропущены)
   - ☐ **Проводить созданные документы** — оставь выключенным на первый раз (документы будут записаны но не проведены — ты сможешь их проверить и провести вручную)
3. Нажми **Проверить соединение** → в логе должно появиться `✓ Соединение установлено`
4. Нажми **Сохранить настройки** → при следующем открытии значения подтянутся автоматически

### Шаг 2. Загрузка

1. В дашборде убедись что есть накладные в статусе **"Обработан"** (синяя плашка) — они и будут загружены
2. В 1С нажми кнопку **Загрузить накладные**
3. В логе появится:
   ```
   [19:45:02] --- Начало загрузки накладных ---
   [19:45:02] Время: 04.04.2026 19:45:02
   [19:45:15] ✓ Загружено накладных: 5
   [19:45:15] Созданные документы:
   [19:45:15]   - Приходная накладная 00-000123 от 24.03.2026
   [19:45:15]   - Приходная накладная 00-000124 от 24.03.2026
   ...
   [19:45:15] --- Загрузка завершена ---
   ```
4. В дашборде у этих накладных статус поменяется на **"Отправлен в 1С"** (зелёная плашка)
5. В 1С открой список **Приходных накладных** — там будут созданные документы

### Шаг 3. Проверка и проведение

1. Для каждой новой накладной в 1С:
   - Открой документ
   - Проверь контрагента (создался ли с правильным ИНН, если API его распознал)
   - Проверь товары — названия, количество, цены
   - Если всё ок → нажми **Провести и закрыть**

### Что делать если документ плохой (нужно пересоздать)

1. В 1С **удали** документ (пометка на удаление → обработка удаления)
2. В дашборде scan.magday.ru найди эту накладную (она в статусе "Отправлен в 1С")
3. Открой её → в адресной строке скопируй ID
4. Сделай API-запрос (или через Postman/curl):
   ```bash
   curl -X POST -H "X-API-Key: your-secret-api-key" \
     http://scan.magday.ru/api/invoices/ID/reset
   ```
   Это вернёт статус в "Обработан" и накладная снова попадёт в следующую загрузку
5. Ещё раз нажми "Загрузить накладные" в 1С

## Часть 3. Что делает обработка под капотом

```
1. GET http://scan.magday.ru/health
   → проверяет что сервис доступен

2. GET http://scan.magday.ru/api/invoices/pending
   → получает список всех накладных со статусом "processed"
   → возвращает массив со всеми данными (id, номер, дата, поставщик, ИНН, items[])

3. Для каждой накладной:
   a) Поиск контрагента:
      - сначала по ИНН (если пришёл)
      - потом по точному имени
      - потом нечётким поиском (LIKE)
      - если не найден — создаёт нового с именем и ИНН

   b) Для каждого товара:
      - поиск номенклатуры по имени (точному, потом нечёткому)
      - если не найдена и галка "Создавать номенклатуру" включена — создаёт новую

   c) Создаёт документ "Приходная накладная" (или "ПоступлениеТоваровУслуг"
      в зависимости от конкретной сборки УНФ):
      - Дата = invoice_date из API
      - Контрагент = найденный/созданный
      - НомерВходящегоДокумента = invoice_number
      - Табличная часть "Запасы" (или "Товары" / "ТоварыУслуги"):
        для каждой строки — Номенклатура, Количество, Цена, Сумма, ЕдиницаИзмерения

   d) Записывает документ (с проведением если галка включена)

4. POST http://scan.magday.ru/api/invoices/:id/confirm
   → помечает накладную как загруженную в 1С (status = sent_to_1c)
```

## Возможные проблемы и решения

### "Не найден подходящий тип документа поступления"
Твоя сборка УНФ использует нестандартное имя документа. Открой 1С в режиме 1С:Предприятие → Документы → найди документ для прихода товаров → посмотри его имя в Конфигураторе → добавь его в функцию `СоздатьПриходнуюНакладную` модуля объекта.

### "Реквизит ИНН не найден в Контрагентах"
Это не ошибка — предупреждение перехватывается. Контрагент будет создан без ИНН (только по имени). Если ИНН важен — нужно найти правильное имя реквизита в твоей УНФ и поправить строку `НовыйКонтрагент.ИНН = ИНН;`.

### Ошибка SSL (https не подключается)
Обработка использует `ЗащищенноеСоединениеOpenSSL()`. В старых 1С возможны проблемы с современными TLS. Используй HTTP адрес: `http://scan.magday.ru` — он тоже работает и проксируется правильно.

### "Код ответа: 401"
Неправильный API-ключ. Зайди в [настройки дашборда](http://scan.magday.ru/#/settings) и сверь ключ.

### Накладные загружаются, но статус в дашборде не меняется
Это значит `POST /confirm` не проходит. Проверь в логе 1С нет ли ошибки подтверждения. Если есть — проверь что 1С может делать POST-запросы (может быть заблокировано firewall).

### Автозагрузка по расписанию
Если хочешь чтобы обработка запускалась сама раз в N минут:
1. Зарегистрируй обработку как **Дополнительная обработка**:
   Администрирование → Печатные формы, отчёты и обработки → Дополнительные отчёты и обработки → Добавить → указать `.epf` → Размещение: любое удобное
2. Создай **Регламентное задание** которое вызывает метод `ЗагрузитьНакладные` этой обработки

## Чеклист перед первым запуском в продакшене

- [ ] `.epf` собран в Конфигураторе и открывается в 1С
- [ ] `Проверить соединение` → ✓ Соединение установлено
- [ ] Выбрана правильная организация и склад
- [ ] Галка "Проводить документы" **выключена** для первого теста
- [ ] В дашборде есть хотя бы одна накладная в статусе "Обработан"
- [ ] После нажатия "Загрузить" в 1С появились документы приходных накладных
- [ ] В дашборде статус изменился на "Отправлен в 1С"
- [ ] Документ в 1С открывается, контрагент и товары заполнены
- [ ] Документ проводится без ошибок (вручную)

Когда весь чеклист пройден — включай галку "Проводить документы" и грузи всё автоматом.
