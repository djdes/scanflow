# Внешняя обработка для 1С:УНФ 1.6

## Загрузка накладных из сервиса JPGExchange

### Описание

Внешняя обработка `.epf` для автоматической загрузки распознанных накладных из веб-сервиса JPGExchange и создания документов "Приходная накладная" в 1С:УНФ 1.6.

### Возможности

- ✅ Получение списка необработанных накладных по HTTP API
- ✅ Автоматический поиск контрагента по названию (нечёткий поиск)
- ✅ Автоматический поиск номенклатуры по названию
- ✅ Создание новой номенклатуры, если не найдена
- ✅ Создание документа "Приходная накладная" с заполненной табличной частью
- ✅ Подтверждение загрузки (отметка в сервисе что накладная обработана)
- ✅ Ручной и автоматический режим (по расписанию)

---

## Создание обработки в Конфигураторе

### Шаг 1: Создать внешнюю обработку

1. Открыть 1С:Предприятие в режиме **Конфигуратор**
2. Меню: **Файл → Новый → Внешняя обработка**
3. Имя: `ЗагрузкаНакладныхJPGExchange`
4. Синоним: `Загрузка накладных из JPGExchange`

### Шаг 2: Добавить реквизиты обработки

В дереве метаданных обработки добавить реквизиты:

| Имя | Тип | Описание |
|-----|-----|----------|
| `АдресСервиса` | Строка(200) | URL сервиса (http://localhost:8899) |
| `КлючAPI` | Строка(100) | API-ключ для авторизации |
| `Организация` | СправочникСсылка.Организации | Организация для документов |
| `Склад` | СправочникСсылка.Склады | Склад поступления |
| `СоздаватьНоменклатуру` | Булево | Создавать новую номенклатуру автоматически |
| `ПроводитьДокументы` | Булево | Проводить созданные документы |

### Шаг 3: Создать форму

1. Правой кнопкой на "Формы" → Добавить → Форма
2. Тип формы: Форма обработки
3. Имя: `ФормаОбработки`

### Шаг 4: Разместить элементы на форме

**Группа "Настройки подключения":**
- Поле ввода: АдресСервиса
- Поле ввода: КлючAPI (пароль)

**Группа "Параметры загрузки":**
- Поле выбора: Организация
- Поле выбора: Склад
- Флажок: СоздаватьНоменклатуру
- Флажок: ПроводитьДокументы

**Команды:**
- Кнопка: "Загрузить накладные" → КомандаЗагрузить
- Кнопка: "Проверить соединение" → КомандаПроверитьСоединение
- Кнопка: "Сохранить настройки" → КомандаСохранитьНастройки

**Поле вывода:**
- Текстовый документ: ПолеЛога (для вывода результатов)

### Шаг 5: Скопировать код модуля формы

Скопировать содержимое файла `МодульФормы.bsl` в модуль формы обработки.

### Шаг 6: Скопировать код модуля объекта

Скопировать содержимое файла `МодульОбъекта.bsl` в модуль объекта обработки.

### Шаг 7: Сохранить как .epf

1. Меню: **Файл → Сохранить как...**
2. Тип файла: **Внешние обработки (*.epf)**
3. Имя: `ЗагрузкаНакладныхJPGExchange.epf`

## Установка в 1С:УНФ 1.6

### Способ 1: Ручной запуск

1. Открыть 1С:УНФ в режиме **Предприятие**
2. Меню: **Файл → Открыть** (или F11)
3. Выбрать файл `ЗагрузкаНакладныхJPGExchange.epf`
4. Заполнить настройки, нажать "Загрузить накладные"

### Способ 2: Регистрация как дополнительная обработка

1. Открыть: **Компания → Ещё больше возможностей → Администрирование**
2. Перейти: **Печатные формы, отчёты и обработки → Дополнительные отчёты и обработки**
3. Нажать: **Добавить из файла...**
4. Выбрать: `ЗагрузкаНакладныхJPGExchange.epf`
5. Настроить публикацию: "Используется"

После регистрации обработка будет доступна в меню **Компания → Дополнительные отчёты и обработки**.

## Настройка сервиса JPGExchange

### API эндпоинты

Обработка использует следующие эндпоинты:

| Метод | URL | Описание |
|-------|-----|----------|
| GET | `/api/invoices/pending` | Получить необработанные накладные |
| GET | `/api/invoices/:id` | Получить детали накладной с товарами |
| POST | `/api/invoices/:id/confirm` | Подтвердить загрузку в 1С |
| GET | `/health` | Проверка доступности сервиса |

### Заголовки

```
X-API-Key: ваш-api-ключ
Content-Type: application/json
```

### Формат данных накладной

```json
{
  "id": 123,
  "invoice_number": "ВМ-105",
  "invoice_date": "2026-01-21",
  "supplier": "ООО \"Вкусный мир ТК\"",
  "total_sum": 18317.19,
  "items": [
    {
      "original_name": "Крупа Гречневая 5кг",
      "mapped_name": "Крупа гречневая",
      "quantity": 10,
      "unit": "шт",
      "price": 150.00,
      "total": 1500.00
    }
  ]
}
```

## Логика работы обработки

### 1. Получение списка накладных

```
GET /api/invoices/pending
→ Список накладных со статусом "processed"
```

### 2. Для каждой накладной

```
GET /api/invoices/{id}
→ Полные данные с товарами
```

### 3. Поиск/создание контрагента

```bsl
// Поиск по наименованию
Контрагент = Справочники.Контрагенты.НайтиПоНаименованию(Поставщик);

// Если не найден — создать
Если Контрагент.Пустая() Тогда
    НовыйКонтрагент = Справочники.Контрагенты.СоздатьЭлемент();
    НовыйКонтрагент.Наименование = Поставщик;
    НовыйКонтрагент.ЮрФизЛицо = Перечисления.ЮрФизЛицо.ЮрЛицо;
    НовыйКонтрагент.Записать();
КонецЕсли;
```

### 4. Создание документа

```bsl
Документ = Документы.ПриходнаяНакладная.СоздатьДокумент();
Документ.Дата = ДатаНакладной;
Документ.Организация = Организация;
Документ.Контрагент = Контрагент;
Документ.Склад = Склад;
```

### 5. Заполнение товаров

```bsl
Для Каждого Товар Из Товары Цикл
    НоваяСтрока = Документ.Запасы.Добавить();
    НоваяСтрока.Номенклатура = НайтиНоменклатуру(Товар.mapped_name);
    НоваяСтрока.Количество = Товар.quantity;
    НоваяСтрока.Цена = Товар.price;
    НоваяСтрока.Сумма = Товар.total;
КонецЦикла;
```

### 6. Подтверждение

```
POST /api/invoices/{id}/confirm
→ Отметка что накладная загружена в 1С
```

## Особенности 1С:УНФ 1.6

### Документ "Приходная накладная"

В разных версиях УНФ документ может называться:
- `ПриходнаяНакладная`
- `ПоступлениеТоваровУслуг`
- `ПоступлениеЗапасов`

**Проверить в конфигураторе:** Документы → найти документ поступления товаров

### Табличная часть

Может называться:
- `Запасы`
- `Товары`
- `ТоварыУслуги`

### Справочник номенклатуры

- `Номенклатура` — основной справочник товаров
- Может быть иерархический (с группами)

### Справочник контрагентов

- `Контрагенты` — поставщики и покупатели
- Реквизит `ЮрФизЛицо` — тип контрагента

## Решение типичных проблем

### Ошибка "Метод не найден"

Проверьте имена документов и справочников в вашей конфигурации. Они могут отличаться от стандартных.

### Ошибка HTTP-соединения

1. Проверьте доступность сервиса: `http://localhost:8899/health`
2. Проверьте правильность API-ключа
3. Если сервер на другом компьютере — проверьте firewall

### Номенклатура не найдена

1. Включите опцию "Создавать номенклатуру автоматически"
2. Или добавьте маппинги в веб-дашборде JPGExchange

### Документ не проводится

Проверьте заполнение обязательных реквизитов:
- Организация
- Склад
- Контрагент
- Валюта (если требуется)
