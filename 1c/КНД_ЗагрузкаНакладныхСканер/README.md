# КНД_ЗагрузкаНакладныхСканер

Внешняя обработка для 1С:УНФ 1.6 — загружает распознанные приходные накладные со `scanflow.ru` в 1С. Создаёт документы `ПриходнаяНакладная`, автоматически находит/создаёт контрагентов (по ИНН и имени) и номенклатуру.

Структура папок — ровно такая же как у существующих обработок `КНД_ЗагрузкаНовыхТоваров` и `КНД_ОбменССайтом_Инвентаризация` в `C:\www-Oleg\`.

## Структура

```
КНД_ЗагрузкаНакладныхСканер/
├── КНД_ЗагрузкаНакладныхСканер.xml          # Корневые метаданные (UUID, имя, форма)
└── КНД_ЗагрузкаНакладныхСканер/
    ├── Ext/
    │   └── ObjectModule.bsl                  # Основной код (~570 строк)
    └── Forms/
        ├── Форма.xml                         # Метаданные формы
        └── Форма/
            └── Ext/
                ├── Form.xml                  # Layout формы (кнопка "Загрузить накладные")
                └── Form/
                    └── Module.bsl            # Модуль формы (обёртка команды)
```

## Что делает

1. `GET scanflow.ru/api/invoices/pending` — получает все накладные в статусе `processed`
2. Для каждой накладной:
   - Находит/создаёт контрагента: сначала по ИНН, потом по точному имени, потом по нечёткому поиску, иначе создаёт нового с ИНН и признаком `Поставщик`
   - Создаёт документ `Документы.ПриходнаяНакладная`:
     - `ДатаВходящегоДокумента` = дата из API
     - `Организация` = `Справочники.Организации.ОсновнаяОрганизация`
     - `Контрагент` = найденный/созданный
     - `НомерВходящегоДокумента` = номер накладной
     - `СтруктурнаяЕдиница` = "Магазин" (или "Основной склад" как fallback)
     - `Комментарий` = `#Загружено из scanflow.ru, ID: {id}`
     - `СуммаДокумента` = `total_sum` из API
   - Заполняет табличную часть `Запасы`: для каждого товара — `Номенклатура`, `Количество`, `Цена`, `Сумма`, `ЕдиницаИзмерения`
   - **Записывает без проведения** (пользователь проверит и проведёт вручную)
3. `POST scanflow.ru/api/invoices/{id}/confirm` — помечает накладную как загруженную

## Конфигурация

Адрес сервиса и API-ключ хардкодятся в **константах в начале модуля** — функции `КОНСТ_АдресСервиса()`, `КОНСТ_Порт()`, `КОНСТ_ИспользоватьSSL()`, `КОНСТ_КлючAPI()`. По умолчанию:

```bsl
scanflow.ru : 443  (HTTPS, с SSL)
API key: <зашит в Ext/ObjectModule.bsl — спросить у админа>
```

Если нужно сменить — открой `Ext/ObjectModule.bsl`, поправь эти четыре функции, пересохрани `.epf`.

## Как собрать .epf из этой папки

Есть **два способа**. Выбери удобный.

### Способ 1 (рекомендуется) — Через "Загрузку конфигурации из файлов"

Этот способ даёт готовый `.epf` автоматически и использует исходники как единый источник правды.

1. Скопируй папку `КНД_ЗагрузкаНакладныхСканер` целиком в удобное место, например в `C:\www-Oleg\` — туда где лежат другие обработки
2. Запусти **1С:Предприятие** → выбери свою базу УНФ → **Конфигуратор**
3. Меню **Файл → Новый... → Внешняя обработка** → **ОК** (создастся пустая обработка)
4. В открывшемся дереве обработки правой кнопкой на корне → **Сравнить, объединить с файлами XML...** (или в меню: Конфигурация → Объединение конфигураций → Из файлов XML)
   - Альтернатива: меню **Конфигурация → Загрузить конфигурацию из файлов XML...** → выбрать корень обработки
5. Укажи путь к файлу `КНД_ЗагрузкаНакладныхСканер.xml` (корневой файл в этой папке)
6. Конфигуратор загрузит структуру: корневые свойства, форму, модуль объекта, модуль формы
7. **Файл → Сохранить как...** → тип `Внешние обработки (*.epf)` → имя `КНД_ЗагрузкаНакладныхСканер.epf`

### Способ 2 — Ручное создание в Конфигураторе

Если способ 1 не получается по какой-то причине — сделай вручную по шагам из существующей инструкции `1c/ПОШАГОВАЯ_ИНСТРУКЦИЯ.md`, но с другим именем и одной кнопкой вместо шести реквизитов:

1. **Файл → Новый → Внешняя обработка**
2. В свойствах корня:
   - Имя: `КНД_ЗагрузкаНакладныхСканер`
   - Синоним: `Загрузка накладных со сканера`
3. **Реквизиты обработки не нужны** — всё зашито в константах модуля
4. Правый клик на `Формы` → **Добавить** → Форма → Управляемая → Назначить основной
5. В конструкторе формы **Вставить**:
   - Одну команду `ЗагрузитьНакладные` с заголовком "Загрузить накладные"
   - Кнопку в командную панель, связанную с этой командой
6. Открыть модуль формы → скопировать содержимое `Forms/Форма/Ext/Form/Module.bsl`
7. Открыть модуль объекта (двойной клик на корне) → скопировать содержимое `Ext/ObjectModule.bsl`
8. **Файл → Сохранить как...** → `КНД_ЗагрузкаНакладныхСканер.epf`

## Как использовать готовый .epf

### Вариант A — Разовый запуск через "Файл → Открыть"

1. 1С:Предприятие (не Конфигуратор) → **Файл → Открыть** → выбрать `КНД_ЗагрузкаНакладныхСканер.epf`
2. Нажать **Загрузить накладные**
3. Проверить в окне сообщений — для каждой накладной будет либо `✓ Создан документ: ...`, либо `✗ Ошибка: ...`
4. Открыть список **Документов → Приходные накладные** → проверить созданные → провести вручную

### Вариант B — Регистрация как "Дополнительная обработка" (для повторных запусков)

В УНФ 1.6 меню — **НСИ и Администрирование → Печатные формы, отчёты и обработки → Дополнительные отчёты и обработки**

1. Нажать **Добавить из файла** → выбрать `.epf`
2. В карточке дополнительной обработки:
   - **Размещение:** выбрать раздел где будет кнопка (обычно "Администрирование" или "Закупки")
   - **Быстрый доступ:** добавить пользователей которым будет видна обработка
3. Записать и закрыть
4. Теперь в указанном разделе появится пункт меню **Загрузить накладные со сканера**

### Вариант C — Регламентная автозагрузка

Для автозагрузки раз в N минут без вмешательства пользователя:

1. Зарегистрировать `.epf` как дополнительную обработку (вариант B)
2. Создать **Регламентное задание**, которое вызывает процедуру `ВыполнитьКоманду("КНД_ЗагрузкаНакладныхСканер", Неопределено)` обработки
3. Задать расписание (раз в 15/30/60 минут)

## Проверка работы

После запуска обработки — зайди в [дашборд](http://scanflow.ru/#/invoices): у загруженных накладных статус должен смениться с **"Обработан"** (синий) на **"Отправлен в 1С"** (зелёный).

## Что делать если загрузка пошла не так

1. **Удалить документ в 1С** (пометка на удаление → обработка удаления)
2. **Сбросить статус накладной на сервере** — через curl:
   ```bash
   curl -X POST -H "X-API-Key: your-secret-api-key" \
     http://scanflow.ru/api/invoices/ID/reset
   ```
3. В дашборде накладная снова будет в статусе "Обработан" и попадёт в следующую загрузку

Ошибки обработки пишутся в **Журнал регистрации 1С** (Администрирование → Журнал регистрации → Имя события "Загрузка накладных со сканера").

## Архитектурные заметки

- Ширина блока `Запасы` — 14 колонок. Мы заполняем только 5 обязательных: Номенклатура, Количество, Цена, Сумма, ЕдиницаИзмерения. Остальные (НДС, Характеристика, Партия, Содержание, Всего, СтавкаНДС, СуммаНДС, Заказ, Себестоимость) — либо рассчитываются при проведении, либо заполняются вручную
- **Не проводим документ** автоматически потому что автопроведение без проверки цен/сумм может создавать проблемы в учёте. Пользователь проверяет каждый документ и проводит вручную
- **Поиск контрагента по ИНН** — самый надёжный способ, потому что имена могут отличаться (ОCR на разных накладных одного поставщика читает имя по-разному). `canonicalizeSupplierName` на сервере приводит "Общество с ограниченной ответственностью", "ООО", "МС ЛОГИСТИК ООО" к единому виду `ООО "МС ЛОГИСТИК"`, а 1C дополнительно ищет по ИНН
- **Создание номенклатуры** — упрощённое: только Наименование + Тип=Запас + ЕдиницаИзмерения=шт. Если товар важный — после загрузки откройте его в 1С и заполните категорию, группу, доп. реквизиты
- **SSL/TLS** — по умолчанию `HTTP://scanflow.ru:80`. Старые версии 1С могут спотыкаться на современном TLS. Если хочешь HTTPS — поменяй `КОНСТ_ИспользоватьSSL()` на `Истина` и `КОНСТ_Порт()` на `443`
